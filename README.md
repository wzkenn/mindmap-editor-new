# 心境圖譜 · 可編輯概念圖与思维导图

依据单张 JPG 图片重建的网页端编辑器，同时提供“图谱”和“脑图”两种视图。图谱是语义内容的基准来源，脑图由同一语义模型实时投影生成，而不是另一份手工维护的数据。

## 本地运行

```bash
pnpm install
pnpm run dev
```

构建生产版本：

```bash
pnpm run build
```

## 已实现

- 一个语义模型、图谱 / 脑图两种视图
- 任一视图修改文字，另一视图自动同步
- 任一视图新增或删除语义主题，另一视图同步增删
- 图谱坐标、图片和装饰与脑图布局分别保存，不互相污染
- 脑图支持子主题、同级主题、左右平衡和向右结构
- 默认“原图结构”按纵向阶段还原：体用分化、事理汇聚、三者互照、法界递进
- 支持汇聚关系、成员关系、递进关系和虚线双向互照，不再把原图强行压成普通树
- `Tab` 新增子主题、`Enter` 新增同级主题
- 自动整理布局、分支折叠、层级大纲和三套配色主题
- 双击画布文字直接编辑
- 文字编辑时 Backspace / Delete 不会误删整个节点
- 通过右侧属性面板编辑文字、字号、字重和对齐
- 拖拽节点与等比例缩放线稿
- 从节点四周圆点创建新连线
- 编辑连线虚实与单/双向箭头
- 脑图模式可新增自由文字、分组框、本地图片和自定义关系线
- 自由元素在语义文字修改、布局切换和图谱 / 脑图切换后仍会保留
- 图层定位、撤销、重做、删除和重置
- 原图半透明底稿开关
- 浏览器统一存档，同时保存语义模型与两种视图状态
- 导入/导出可编辑 JSON、导出 PNG

## 技术选择

核心采用 [React Flow / xyflow](https://github.com/xyflow/xyflow)，借用了它成熟的节点、连线、缩放、选择与 JSON 状态结构。自定义主题节点与曲线分支让同一套画布既能复刻原始图谱，也能承载层级思维导图。

调研中也比较了：

- [SimpleMindMap](https://github.com/wanglin2/mind-map)：快捷键、结构切换、大纲、主题和导入导出等产品形态很接近 XMind，作为本次脑图交互的主要参考。
- [Mind Elixir](https://github.com/ssshooter/mind-elixir-core)：轻量、框架无关，节点操作和主题能力适合作为脑图交互参考。
- [tldraw](https://github.com/tldraw/tldraw)：自由白板和自定义形状能力完整，但默认产品形态较重，精确复刻需要大幅改造 UI。
- [Excalidraw](https://github.com/excalidraw/excalidraw)：自带撤销、导出和开放 JSON，适合手绘白板，但视觉风格会明显偏向 Excalidraw。
- [react-konva](https://github.com/konvajs/react-konva)：像素级 Canvas 绘制灵活，但节点关系、连接与编辑器状态需要自行搭建。

本项目因此保留 React Flow 作为“关系结构层”，用 DOM/SVG 自定义节点作为“视觉还原层”，并借鉴成熟脑图库的快捷键、自动布局和大纲结构。没有直接替换为完整脑图库，是因为当前产品还需要兼容原图复刻所需的自由节点与非树状连线。

## 数据结构

初始图谱位于 `src/initialData.ts`，其中的文字节点通过 `semanticId` 接入统一语义模型。`src/semanticModel.ts` 负责从图谱生成实体，以及层级、汇聚、互照、成员和递进关系，再把同一模型分别投影成自由图谱与语义脑图。`src/mindMapData.ts` 只提供脑图节点和关系线的视图组件，不再保存独立的内容副本。

JSON 文件格式版本为 `2`，同时包含：

- `semanticModel`：唯一的文字、说明与父子关系来源
- `views.diagram`：图谱坐标、素材和装饰状态
- `views.mindmap`：脑图坐标、折叠、主题和布局状态
